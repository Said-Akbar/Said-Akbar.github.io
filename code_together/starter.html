html course notes
  • Always make sure your html follows DOM structure standards. e.g. html>head>body
  • your web site should be accessible for disabled: no flashing photos, no colorful content, follows standards of W3C, images have alternative text
  • use w3 validator to validate your html code
  • use Wave tool to understand if your page meets accessibility requirements
  • refer to mozilla MDN docs to read on different html tags and attributes
  • Responsive design means the website can change its view to be mobile friendly. CSS allows those features
  • when you type a web address, DNS looks up the IPv6 address (location) of your website and then web server returns files and your browser shows them as a web page
  • host your free website here : byet host

CSS3 course notes
  • design first. Code second. Design in paper
  • don't spend more than 20 minutes figuring out the problem. Ask others or search in google
  • CSS is a set of rules for styling html elements. General syntax: selector {property:value;}
  • always use semi-colons and closing brackets
  • style tag is defined in head tag of html after title
  • instead of inline/internal css, we should use external css to make changes easier. < link rel="stylesheet" href="style.css">
  • order of cascading rules: default, external, internal and inline. Browser will override your css if inline exists.
  • rule of precedence for styling: always last line or last imported css will have high priority
  • you can use !important next to value of css to make it priority no matter if other styling for the same element exists or not
  • always separate your styling from html
  • css example that changes the same html into different designs: css zen garden
  • color name conventions: hex, rbg
  • test your site with contrast checker with the Wave tool
  • font-variant: you can select all uppercase with this property
  • font-size: use percentages to scale font to the display
  • selectors can select multiple elements with a comma
  • align-text to left, center, right, bottom
  • display elements: inline (next to other element), block (takes all horizontal width)
  • inline-block: same as inline but accepts height and width; none - remove from page
  • example:
    span A B C
  • float repositions elements to right or left within its own section. Elements will not overlap
  • clear: keep floating elements away for the current element
  • use overflow to fit the content
  • overflow: show, hidden, scroll, auto
  • we can also use table-cell to correctly format elements

  • visibility: hidden is different from display: none. The former takes up the hidden space while the latter completely removes the element.
  • Box model. Any element can have a border around it. Border style must be specified. Mostly we use solid border
  • border-width: 1px 2px 3px 4px; specifies the border width for an element in a clockwise order for sides. border-width: 1px sets 1px to all sides
  • margin: additional space outside the border
  • padding: empty space between text and element
  • width of a box is width+2*(padding+margin+border)
  • use margin: 0 auto to center element horizontally
  • measures: absolute: px, mm, pt; Fluid: %, em, rem
  • use box model to reduce page complexity

  • text-decorator. links
  • links have states: a:visited, a:hover,a:focus, a:active
  • precedence: link>hover>visited/active
  • a:focus{property: value}
  • there are tools that help creating css styling easier: css generator or google "developer tools"

  • advanced selectors:
    1. descendant selector (nav a) - selects anchors inside nav;
    2. child selector (nav>a) - immediately after nav, style all anchors. If there is a paragraph inbetween nav and a, no styling applies
    3. adjacent sibling: nav+a - elementmust be at the same level and follow each other
    4. id selector: use pound. Developers are moving out of id usage.
    5. class selector: .class. you have to specify all your similar tags with your class name
    6. narrowing the scope: p.main - all paragraphs using main class
    7. you can combine scope with different elements using comma: p.main, .classes, #ids
    8. * - selects all elements
    9. attribute selector: a[href='info.html'] this selects all info.html links
    10. pseudo-classes, pseudo-elements
  • attribute selectors with matching operators: a[href^='http://umich'] match the beginning
  • match the end: img[src$='.png']
  • * wildcard: a[href*='umich']
  • useful css tricks: link
  • some css does not work in all browsers. We can use browser prefixes to support our css styling
  • check https://caniuse.com for supported properties
  • for prefixes, you are just adding prefix and property e.g. -o-column-count for opera column-count; -webkit-for Android, chrome, IOS, safari; -o- opera; -ms-internet explorer;-moz- firefox; moz.
  • In your browser, you can inspect and change what looks good, then copy that to your code
  • design for accessibility: follow POUR: Perceivable, Operable, understandable, Robust
  • make the page easy to navigate. do not overcrowd the page.
  • you want a clean web page that conveys your information

    • Pseudo-classes

  • pseudo-class: elements populated dynamically. a:hover{}
  • positional pseudo-classes. gets the position and applies style: li:first-child{}, li:nth-child(4)
  • textual :first-letter

    • Transitions

  • transitions alter the appearance of the element if you hover over
  • transition-propertywhat you want to change?
  • transition-durationhow long should it take
  • transition-timing
  • transition-delay
  • example. for transitions you always need start and end state. end state can be :hover, :click etc.
    hover over or click
  • for accessibility purposes, do not make site functions dependent on transitions

    • transforms

  • transform changes the appearance of elements
  • translate moves element in x and y axis
  • rotate - spins element
  • scale - make wider or taller
  • skew - skews the element
  • matrix - combines above 4 properties in one
  • transform used in combination with state changes
